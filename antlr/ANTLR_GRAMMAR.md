# RiScript - Dual Parser Implementation (Chevrotain & ANTLR)

This project includes support for both **Chevrotain** and **ANTLR** parsers for the RiScript language.

## Overview

- **Default Parser**: Chevrotain (currently used in main codebase)
- **Alternative Parser**: ANTLR 4.13.0 (available in `antlr/js/`)
- **Tests**: Run with either parser to ensure compatibility

## Running Tests

### Test with Chevrotain (Default)
```bash
npm test
```

### Test with ANTLR
```bash
USE_ANTLR=true npm test
```

## Architecture

### Chevrotain Implementation
- **Files**: `src/parser.js`, `src/visitor.js`, `src/tokens.js`
- **Type**: CST-based (Concrete Syntax Tree)

### ANTLR Implementation
- **Files**: `antlr/js/*.js` (generated) + `antlr/js/AntlrRiScriptVisitor.js` (custom)
- **Grammar**: `antlr/riscript.g4`
- **Type**: Parse tree-based

## Generated ANTLR Files

The following files are generated by ANTLR from `riscript.g4`:

- `antlr/js/riscriptLexer.js` - Tokenizer
- `antlr/js/riscriptParser.js` - Parser
- `antlr/js/riscriptVisitor.js` - Base visitor (default implementation)
- `antlr/js/riscript.tokens` - Token definitions
- `antlr/js/riscript.interp` - Interpreter data

## Custom ANTLR Visitor

`antlr/js/AntlrRiScriptVisitor.js` extends the ANTLR-generated visitor to provide RiScript semantics:

- Resolves symbols from context
- Evaluates choices
- Applies transforms
- Handles assignments (static and dynamic)
- Supports entity decoding

## Usage

### Using Chevrotain (Recommended)
```javascript
import { RiScript } from './src/index.js';

const ri = new RiScript();
const result = ri.evaluate('$greeting', { greeting: 'hello' });
console.log(result); // "hello"
```

### Using ANTLR
```javascript
import { RiScript } from './src/index.js';
import antlr4 from 'antlr4';
import riscriptLexer from './antlr/js/riscriptLexer.js';
import riscriptParser from './antlr/js/riscriptParser.js';
import { AntlrRiScriptVisitor } from './antlr/js/AntlrRiScriptVisitor.js';

const ri = new RiScript();
const input = '$greeting';
const inputStream = new antlr4.InputStream(input);
const lexer = new riscriptLexer(inputStream);
const tokenStream = new antlr4.CommonTokenStream(lexer);
const parser = new riscriptParser(tokenStream);
const tree = parser.script();

const visitor = new AntlrRiScriptVisitor(ri, { greeting: 'hello' });
const result = visitor.start({ tree, input });
console.log(result); // "hello"
```

## Regenerating ANTLR Files

To regenerate the ANTLR parser after modifying `riscript.g4`:

```bash
java -jar ./antlr/antlr-4.13.0-complete.jar -Dlanguage=JavaScript -visitor -o ./antlr/js antlr/riscript.g4
```

## Notes

- Both parsers implement the same grammar and should produce equivalent results
- Chevrotain is currently the primary implementation
- ANTLR implementation is also functional
- The custom ANTLR visitor (`AntlrRiScriptVisitor.js`) handles RiScript semantics

## Dependencies

- `chevrotain` - Main parser library (npm installed)
- `antlr4` - ANTLR runtime for JavaScript (npm installed)
- `he` - HTML entity decoder
- `mingo` - Query library for gates
